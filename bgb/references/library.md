# bgb library & paths

How every bgb skill finds your games and their files. A skill that touches game
data should resolve the rules root (below) before reading or writing anything.

## The rules root (one config)

All games live under a single user-chosen folder — the **rules root**. Its path is
stored in a small config file, the only thing bgb keeps under `~/.claude/`:

- Config file: `~/.claude/bgb-games/config.json`
- Shape: `{ "rulesRoot": "<absolute path to your rules folder>" }`

Use Bash to read it so `~` expands (e.g. `cat ~/.claude/bgb-games/config.json`).
On Windows the config lives at `%USERPROFILE%\.claude\bgb-games\config.json`.

### Resolving the rules root (do this first)

1. Read `config.json`. If it exists and `rulesRoot` is a non-empty path that still
   exists on disk, use it.
2. If the config is missing, empty, or its `rulesRoot` no longer exists, **ask the
   user once**: "Where's your board-game rules folder? (the folder that holds one
   subfolder per game)". Take the path they give, verify it exists, then write
   `~/.claude/bgb-games/config.json` containing `{ "rulesRoot": "<that path>" }`
   (create the `~/.claude/bgb-games/` directory first if needed). Don't ask again on
   later calls.
3. `/bgb:setup` is the explicit command to set or change the root at any time.

## One folder per game

Under the rules root, each game is its own subfolder holding **everything** for that
game — your rulebook PDFs and the files bgb generates:

```
<rulesRoot>/
  Wingspan/
    <anything>.pdf       ← your official rulebook PDF(s); any filename
    strategy.md          ← generated by /bgb:new, revised by /bgb:research
    insights.md          ← generated/appended by /bgb:insights
    rules-digest.md      ← generated by /bgb:rule (or /bgb:new) from the PDF
  Terraforming Mars/
    ...
```

The generated files use **fixed names** inside the game's folder — the folder itself
identifies the game, so no game prefix is needed:
`strategy.md`, `insights.md`, `rules-digest.md`.

### Finding a game's folder

Match the game the user named to a subfolder under the rules root:

1. List the subfolders (`ls "<rulesRoot>"`).
2. Normalize for matching — lowercase, then drop spaces, hyphens, underscores, and
   punctuation — and compare the requested name to each folder name. A close match
   wins (e.g. "terraforming mars" ↔ `Terraforming-Mars` ↔ `terraforming_mars`).
3. **No match:** list the available folders and ask which one — or, for a brand-new
   game (`/bgb:new`), create a new subfolder named with the game's natural name
   (e.g. `Terraforming Mars`).
4. **Ambiguous** (more than one plausible match): show the candidates and ask.

When you create or write a game's files, put them inside that game's folder using
the fixed names above. **Never write game data into the plugin directory.**
